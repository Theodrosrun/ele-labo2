3 Horloges Posix

Nous allons repeter l'experience avec les horloges POSIX. Nous allons executer le code gettimeofday3.c qui va executer une boucle qui execute le fonction clock_gettime() sur les quatres horloges POSIX

CLOCK_REALTIME
CLOCK_MONOTONIC
CLOCK_PROCESS_CPUTIME_ID
CLOCK_THREAD_CPUTIME_ID

code gettimeofday3.c:

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <time.h>

#define NB_MESURES 30

const char *posix_clocks[] = {
    "CLOCK_REALTIME",
    "CLOCK_MONOTONIC",
    "CLOCK_PROCESS_CPUTIME_ID",
    "CLOCK_THREAD_CPUTIME_ID"
};

const clockid_t posix_clock_ids[] = {
    CLOCK_REALTIME,
    CLOCK_MONOTONIC,
    CLOCK_PROCESS_CPUTIME_ID,
    CLOCK_THREAD_CPUTIME_ID
};

int main(int argc, char **argv) {
    struct timespec ts[NB_MESURES];
    int i;

    for (i = 0; i < sizeof(posix_clocks) / sizeof(posix_clocks[0]); ++i) {

        // Get the time with the specified clock and store it in the ts array
        for (int j = 0; j < NB_MESURES; ++j) {
            if (clock_gettime(posix_clock_ids[i], &ts[j]) != 0) {
                perror("clock_gettime failed");
                return EXIT_FAILURE;
            }
        }


        // Write the time to a file with specified clock
        char filename[256];
        snprintf(filename, sizeof(filename), "%s.txt", posix_clocks[i]);
        FILE *fp = fopen(filename, "w");
        if (fp == NULL) {
            perror("Failed to open file");
            return EXIT_FAILURE;
        }

        for (int j = 0; j < NB_MESURES; ++j) {
            fprintf(fp, "%2d : %ld.%09ld\n", j, ts[j].tv_sec, ts[j].tv_nsec);
        }

        fclose(fp);
    }

    return EXIT_SUCCESS;
}

Afin d'analyser au mieux les differences, nous avons calculé la différence entre deux valeurs consécutives dans les deux fichiers puis établis nous avons tracé le résultat dans un graphique.

En analysant les resultats, on voit que, contrairement à la documentation de la méthode clock_gettime() obtenu sur man clock_gettime

DESCRIPTION
...

           struct timespec {
               time_t   tv_sec;        /* seconds */
               long     tv_nsec;       /* nanoseconds */
           };
...

les resultat onbtenus ont une granularité de 10[ns] alors que la précision est de l'ordre de la [ns].
Il apparait alors que le processeur présent sur la DE1-SoC n'est pas capable de mesurer le temps avec une granularité de 1[ns].

En effet, si l'on se réfère à la frequqence du cpu qui est de 980[MHz], on peut calculer la granularité de la mesure de temps avec la formule suivante:

granularite = 1 / 980e6
granularite = granularite * 1e9 # to get the result in [ns]
print(round(granularite,2), " [ns]")

4. Conclusion

Noter que, quand bien meme les horloges POSIX sont plus précises que gettimeofday(), elles ne sont pas capable de mesurer le temps avec une granularité de 1[ns] selon le CPU embarqué.
